## ğŸ’» Development Guidelines

### Code Style

- **JavaScript**: Use ES6+ features, maintain consistent formatting
- **CSS**: Follow Tailwind CSS utility-first approach
- **HTML**: Use semantic HTML5 elements
- **Comments**: Add JSDoc comments for functions and modules

### Best Practices

1. **Keep it simple** - Prefer clarity over cleverness
2. **Document your changes** - Add comments explaining complex logic
3. **Test locally** - Run `npm run dev` and verify changes work
4. **No console.logs** - Remove debug statements before committing
5. **Maintain consistency** - Follow existing code patterns

### Commit Messages

Use clear, descriptive commit messages:

```
feat: Add new rental pricing calculator
fix: Resolve mobile menu toggle issue
docs: Update API documentation
style: Improve button hover states
refactor: Simplify form validation logic
```
